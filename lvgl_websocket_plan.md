# ESP32 LVGL9 WebSocket 프로젝트 구현 계획서 (개정판)

하드웨어 드라이버(TFT_eSPI, XPT2046) 및 핀 설정(IO34 ADC, Touch SPI)을 프로젝트 계획에 반영하였습니다.

## 1. 하드웨어 사양 반영
*   **디스플레이**: 320x240 해상도 (Landscape), `TFT_eSPI` 사용.
*   **터치**: `XPT2046_Touchscreen` (CS:33, IRQ:36).
*   **ADC**: IO34 (12bit 읽기 후 0-255 매핑).

## 2. UI 구성 (LVGL 9.0.0 기반)

### 탭 1: WiFi 스캔 및 연결 제어
1.  **스캔 버튼**: 클릭 시 `WiFi.scanNetworks()`를 실행하여 주변 AP 검색.
2.  **AP 리스트**: 검색된 목록을 `lv_list`로 표시하고, 항목 선택 시 비밀번호 입력을 위한 키보드 표시.
3.  **연결 상태**: 할당받은 디바이스 IP 주소(`WiFi.localIP()`)를 라벨로 명시.
4.  **연결/해제 버튼**: 현재 WiFi 상태에 따라 연결 시도 또는 연결 끊기 수행.

### 탭 2: ADC 모니터링 및 웹소켓 통신
1.  **Round Scale (Meter)**: IO34 핀의 ADC 값을 0~255 범위의 바늘 계측기로 시각화.
2.  **수치 라벨**: 현재 ADC 값을 텍스트로도 병행 표시.
3.  **웹소켓 로그 영역**: 클라이언트로부터 전송된 메시지를 출력하는 스크롤 가능한 텍스트 영역.
4.  **웹소켓 서버 (Port 8000)**:
    *   서버에서 클라이언트로: 실시간 ADC 값 전송.
    *   클라이언트에서 서버로: 텍스트 메시지 수신 후 UI에 로그 기록.

## 3. 웹 클라이언트 (HTML/JS)
*   일축된 단일 HTML 파일로 제작.
*   **기능**: 웹소켓 연결/해제, ADC 데이터 수신 표시, 서버로 메시지 전송 버튼.

## 4. 구현 및 검증 단계
*   **1단계**: `lvgl_generated_claude.ino`의 드라이버 코드를 활용하여 LVGL 9 기본 하드웨어 초기화 코드 작성.
*   **2단계**: 탭뷰 및 위젯 레이아웃 구현.
*   **3단계**: WiFi 스캔 및 웹소켓 서버 로직 통합.
*   **4단계**: 배포 가능한 HTML 클라이언트 파일 생성 및 통합 테스트.
